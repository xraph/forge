# Sessions API

Complete reference for AuthSome Go session management endpoints, covering session lifecycle, device tracking, multi-session support, and advanced security controls.

## Overview

The Sessions API provides comprehensive session management functionality including device tracking, location-based security, session analytics, and multi-device support with granular control over session lifecycle.

### Base URL

```
GET    /auth/sessions
GET    /auth/sessions/current
DELETE /auth/sessions/current
GET    /auth/sessions/{id}
DELETE /auth/sessions/{id}
DELETE /auth/sessions
POST   /auth/sessions/refresh
GET    /auth/sessions/devices
DELETE /auth/sessions/devices/{deviceId}
POST   /auth/sessions/verify-device
```

## List Sessions

Get all active sessions for the current user.

### Endpoint

```http
GET /auth/sessions
```

### Headers

```http
Authorization: Bearer <session_token>
```

### Query Parameters

| Parameter | Type | Description | Default |
|-----------|------|-------------|---------|
| `page` | integer | Page number (1-based) | 1 |
| `limit` | integer | Items per page (1-100) | 20 |
| `status` | string | Filter by status (active, expired, revoked) | active |
| `deviceType` | string | Filter by device type | - |
| `location` | string | Filter by location/country | - |
| `sort` | string | Sort field (createdAt, lastActivity, location) | lastActivity |
| `order` | string | Sort order (asc, desc) | desc |

## Revoke All Sessions

Revoke all sessions except the current one.

### Endpoint

```http
DELETE /auth/sessions
```

### Headers

```http
Authorization: Bearer <session_token>
```

### Query Parameters

| Parameter | Type | Description | Default |
|-----------|------|-------------|---------|
| `excludeCurrent` | boolean | Exclude current session from revocation | true |
| `deviceType` | string | Only revoke sessions from specific device type | - |
| `olderThan` | string | Only revoke sessions older than duration (e.g., "7d", "1h") | - |

## Session Security

### Risk Assessment

Sessions are continuously monitored for security risks:

| Risk Factor | Description | Score Impact |
|-------------|-------------|--------------|
| `new_device` | Login from unrecognized device | +0.3 |
| `new_location` | Login from new geographic location | +0.2 |
| `suspicious_ip` | IP address flagged as suspicious | +0.5 |
| `vpn_proxy` | Connection through VPN or proxy | +0.1 |
| `unusual_time` | Login at unusual time for user | +0.1 |
| `rapid_location_change` | Impossible travel between locations | +0.8 |
| `multiple_failed_attempts` | Recent failed login attempts | +0.4 |

### Device Trust Levels

| Trust Level | Score Range | Description |
|-------------|-------------|-------------|
| `trusted` | 0.8 - 1.0 | Verified and regularly used device |
| `known` | 0.5 - 0.79 | Recognized but not fully trusted |
| `unknown` | 0.2 - 0.49 | New or suspicious device |
| `blocked` | 0.0 - 0.19 | Blocked or compromised device |

### Session Types

| Type | Description | Default Expiry |
|------|-------------|----------------|
| `web` | Browser-based session | 24 hours |
| `mobile` | Mobile app session | 30 days |
| `api` | API access session | 1 hour |
| `admin` | Administrative session | 4 hours |
| `temporary` | Short-lived session | 15 minutes |

## Rate Limiting

Session endpoints have specific rate limits:

| Endpoint | Rate Limit | Window |
|----------|------------|--------|
| `GET /auth/sessions` | 100 requests per user | 1 hour |
| `GET /auth/sessions/current` | 200 requests per user | 1 hour |
| `DELETE /auth/sessions/{id}` | 20 requests per user | 1 hour |
| `DELETE /auth/sessions` | 5 requests per user | 1 hour |
| `POST /auth/sessions/refresh` | 50 requests per user | 1 hour |
| `GET /auth/sessions/devices` | 50 requests per user | 1 hour |
| `DELETE /auth/sessions/devices/{deviceId}` | 10 requests per user | 1 hour |
| `POST /auth/sessions/verify-device` | 10 requests per user | 1 hour |

<Callout type="warning">
  Excessive session management requests may trigger additional security checks and temporary restrictions.
</Callout>
