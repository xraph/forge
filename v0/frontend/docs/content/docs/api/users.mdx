# Users API

Complete reference for AuthSome Go user management endpoints, covering user profiles, preferences, password management, and account settings.

## Overview

The Users API provides comprehensive user management functionality including profile management, password operations, email verification, account preferences, and user search capabilities.

### Base URL

```
GET    /auth/user
PUT    /auth/user
DELETE /auth/user
POST   /auth/user/change-password
POST   /auth/user/change-email
POST   /auth/user/verify-email
GET    /auth/users
GET    /auth/users/{id}
```

### Example Usage

<Tabs items={['JavaScript', 'TypeScript', 'Go', 'cURL']}>
  <Tab value="JavaScript">

    ```javascript
    const response = await fetch('/auth/user', {
    ```

  <Tab value="TypeScript">
    ```typescript
    import { AuthSome, User } from '@authsome/js-sdk';

    const auth = new AuthSome({ baseUrl: '/auth' });

    try {
      const user: User = await auth.getCurrentUser();
      
      console.log('Current user:', user);
      
      // Type-safe access to user properties
      if (user.twoFactorEnabled) {
        console.log('2FA is enabled for this user');
      }
      
      // Access user preferences
      const theme = user.metadata?.preferences?.theme || 'light';
      console.log('User theme preference:', theme);
      
    } catch (error) {
      if (error instanceof AuthSomeError) {
        console.error('Failed to get user:', error.code, error.message);
      }
    }
    ```

  <Tab value="Go">
    ```go
    package main

    import (
        "fmt"
        "net/http"
        "encoding/json"
    )

    func getCurrentUser(sessionToken string) error {
        req, err := http.NewRequest("GET", "/auth/user", nil)
        if err != nil {
            return fmt.Errorf("failed to create request: %w", err)
        }

        req.Header.Set("Authorization", "Bearer "+sessionToken)
        req.Header.Set("Content-Type", "application/json")

        client := &http.Client{}
        resp, err := client.Do(req)
        if err != nil {
            return fmt.Errorf("failed to make request: %w", err)
        }
        defer resp.Body.Close()

        var result map[string]interface{}
        if err := json.NewDecoder(resp.Body).Decode(&result); err != nil {
            return fmt.Errorf("failed to decode response: %w", err)
        }

        if success, ok := result["success"].(bool); ok && success {
            if data, ok := result["data"].(map[string]interface{}); ok {
                if user, ok := data["user"].(map[string]interface{}); ok {
                    fmt.Printf("Current user: %s (%s)\n", user["name"], user["email"])
                    fmt.Printf("2FA enabled: %v\n", user["twoFactorEnabled"])
                }
            }
        } else {
            fmt.Printf("Failed to get user: %v\n", result["error"])
        }

        return nil
    }
    ```

## Rate Limiting

User management endpoints have specific rate limits:

| Endpoint | Rate Limit | Window |
|----------|------------|--------|
| `GET /auth/user` | 100 requests per user | 1 hour |
| `PUT /auth/user` | 10 requests per user | 1 hour |
| `POST /auth/user/change-password` | 3 requests per user | 1 hour |
| `POST /auth/user/change-email` | 3 requests per user | 1 hour |
| `POST /auth/user/verify-email` | 5 requests per IP | 1 hour |
| `DELETE /auth/user` | 1 request per user | 24 hours |
| `GET /auth/users` | 100 requests per user | 1 hour |

<Callout type="warning">
  Password change attempts are limited to prevent brute force attacks. After 3 failed attempts, the user must wait 1 hour before trying again.
</Callout>

## Validation Rules

### Username Validation

- Length: 3-30 characters
- Characters: Letters, numbers, underscores, hyphens
- Case: Case-insensitive uniqueness
- Reserved: Cannot use reserved words (admin, api, www, etc.)

### Password Validation

- Minimum length: 8 characters
- Must contain: uppercase, lowercase, number, special character
- Cannot contain: common passwords, user's email/name
- History: Cannot reuse last 5 passwords

### Email Validation

- Valid email format
- Domain validation
- Disposable email detection (configurable)
- Uniqueness across the organization

## Security Features

### Profile Security

- Email verification required for email changes
- Password confirmation for sensitive operations
- Audit logging for all profile changes
- Rate limiting on sensitive endpoints

### Data Privacy

- Personal data encryption at rest
- GDPR compliance features
- Data export capabilities
- Right to be forgotten (account deletion)

### Account Protection

- Account lockout after failed attempts
- Suspicious activity detection
- Device-based security policies
- Session management and revocation
