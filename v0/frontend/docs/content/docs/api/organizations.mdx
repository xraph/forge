# Organizations API

Complete reference for AuthSome Go organization management endpoints, covering multi-tenant organization creation, membership management, role-based access control, and organization settings.

## Overview

The Organizations API provides comprehensive multi-tenant functionality allowing users to create and manage organizations, invite members, assign roles, and configure organization-specific settings.

### Base URL

```
GET    /auth/organizations
POST   /auth/organizations
GET    /auth/organizations/{id}
PUT    /auth/organizations/{id}
DELETE /auth/organizations/{id}
GET    /auth/organizations/{id}/members
POST   /auth/organizations/{id}/members
PUT    /auth/organizations/{id}/members/{userId}
DELETE /auth/organizations/{id}/members/{userId}
```

## List Organizations

Get all organizations the current user is a member of.

### Endpoint

```http
GET /auth/organizations
```

### Headers

```http
Authorization: Bearer <session_token>
```

### Query Parameters

| Parameter | Type | Description | Default |
|-----------|------|-------------|---------|
| `page` | integer | Page number (1-based) | 1 |
| `limit` | integer | Items per page (1-100) | 20 |
| `role` | string | Filter by user's role in organization | - |
| `status` | string | Filter by organization status | active |
| `sort` | string | Sort field (name, createdAt, memberCount) | createdAt |
| `order` | string | Sort order (asc, desc) | desc |

## Organization Roles

AuthSome Go supports hierarchical role-based access control:

### Default Roles

| Role | Description | Permissions |
|------|-------------|-------------|
| `owner` | Organization owner | All permissions |
| `admin` | Administrator | All except ownership transfer |
| `moderator` | Content moderator | Member management, content moderation |
| `member` | Regular member | Basic organization access |
| `guest` | Limited access | Read-only access |

### Permission System

Permissions follow the format `resource:action`:

| Permission | Description |
|------------|-------------|
| `org:read` | View organization details |
| `org:update` | Update organization settings |
| `org:delete` | Delete organization |
| `members:read` | View member list |
| `members:invite` | Invite new members |
| `members:remove` | Remove members |
| `members:update_roles` | Update member roles |
| `settings:read` | View organization settings |
| `settings:update` | Update organization settings |
| `billing:read` | View billing information |
| `billing:update` | Update billing settings |

## Rate Limiting

Organization endpoints have specific rate limits:

| Endpoint | Rate Limit | Window |
|----------|------------|--------|
| `GET /auth/organizations` | 100 requests per user | 1 hour |
| `POST /auth/organizations` | 5 requests per user | 1 hour |
| `PUT /auth/organizations/{id}` | 10 requests per user | 1 hour |
| `GET /auth/organizations/{id}/members` | 100 requests per user | 1 hour |
| `POST /auth/organizations/{id}/members` | 20 requests per user | 1 hour |
| `PUT /auth/organizations/{id}/members/{userId}` | 10 requests per user | 1 hour |
| `DELETE /auth/organizations/{id}/members/{userId}` | 10 requests per user | 1 hour |

<Callout type="info">
  Organization creation is limited to prevent abuse. Users can create up to 5 organizations per day in SaaS mode.
</Callout>

|------|-------------|-------------|

| `CANNOT_REMOVE_OWNER` | Cannot remove organization owner | 422 |
